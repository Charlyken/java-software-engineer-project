# Système de Gestion des Inscriptions

## À propos du projet

Ce projet est une initiative universitaire développée dans le cadre d'un projet de développement logiciel à l'ESIGELEC. L'objectif est de concevoir et de mettre en œuvre un système robuste pour gérer les inscriptions des étudiants aux dominantes (spécialisations) proposées par l'école.

## Acteurs du système

Le système est conçu pour deux types d'utilisateurs principaux :

*   **Étudiant :** Peut consulter les dominantes disponibles, faire son choix d'inscription et suivre le statut de sa demande.
*   **Administrateur :** Gère l'ensemble du processus, y compris l'ouverture et la fermeture des périodes d'inscription, la validation des choix des étudiants, et l'administration générale du système.

## Contenu du dépôt

Ce dépôt sert actuellement de hub de documentation et de planification pour le projet. Il ne contient pas encore de code source.

Voici la structure actuelle :

*   `./compte_Rendu/` : Contient les comptes rendus des réunions de projet. Ces documents retracent l'historique des décisions, les objectifs fixés et l'avancement de la conception.
*   `./Diagramme de Cas d'utlisation/` : Renferme les artefacts de conception essentiels du système, notamment :
    *   Le diagramme de cas d'utilisation (Use Case) qui décrit visuellement les interactions et les fonctionnalités.
    *   Les spécifications détaillées pour les fonctionnalités clés comme "s'inscrire".
